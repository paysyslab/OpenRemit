## Objective
The Transaction List provides a structured table of all transactions with core details, filters, and a unified action menu.

![authentication flow](/img/txn/transactions.png)

## Columns 
- **Txn ID**
- **Branch Code**
- **MTO / Partner**
- **Type** (OTC / FT / IBFT or configured types)
- **Amounts** (FCY and PKR)
- **Status**
- **Actions** (View Details; other permitted actions)

## How to Search
1. Navigate to **Backoffice → Transactions**.
2. Use available filters:
   - Date range;
   - Partner/MTO;
   - Branch code;
   - Status (Completed, Failed, Pending, In Review).
3. Use search bar for direct lookup by Txn ID or reference.

## Recommended Usage
- Always filter by date range first for performance.
- For recurring issues, filter by partner and status to isolate patterns.
- Use the action menu to open transaction details and validate timeline.
