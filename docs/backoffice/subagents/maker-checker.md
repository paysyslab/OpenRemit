## Objective
Provides controlled onboarding with dual authorization for sub-agent and user requests.

## What Appears Here
- Rejected sub-agent creation requests requiring correction;
- Rejected user creation requests requiring correction;
- Pending requests awaiting Checker approval.

## Maker Actions
- Correct and resubmit requests;
- Provide accurate settlement, branch, and contact mappings.


![SubAgent](/img/subAgents/userMaker.png)

## Checker Actions
- Validate completeness and correctness;
- Approve to activate; reject with reason if needed.

All actions are auditable.
