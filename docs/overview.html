Transaction Mechanisms

Three Ways to Integrate

OpenRemit supports three key transaction mechanisms, each tailored to the specific capabilities and needs of remittance partners — from full API integration to batch file processing.

Push vs Pull Mechanism
01
Push Mechanism
How: Banks push transaction data directly to OpenRemit's standardized APIs for immediate validation and processing.
Benefit: Direct multi-partner engagement without managing disparate systems. No reliance on external triggers.
02
Pull Mechanism
How: OpenRemit pulls transaction data from partner APIs in real time, consolidating and processing centrally.
Benefit: Banks retain full control over transaction flows while supporting numerous remittance services seamlessly.
File-Based Integration
03
File Mechanism

How it works: Partners without API capabilities send day-end batch transaction files. OpenRemit converts them into individual transactions for processing.

Key benefit: No partner is excluded regardless of technical capability. Even smaller remittance partners can fully participate in the network.
